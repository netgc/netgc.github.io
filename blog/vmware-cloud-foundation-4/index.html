VMware Cloud Foundation+

随着 VMware 继续执行转变本地基础架构组合的战略，该战略首次宣布为北极项目在 VMworld 2021 上，6 月推出的 VMware vSphere+ 和 VMware vSAN+ 仅仅是个开始。

介绍 VMware Cloud Foundation+

该战略的下一阶段将通过 VMware Cloud Foundation+ 为您的私有云部署提供云的完整堆栈优势。VMware Cloud Foundation+ 将添加一个新的云连接架构，用于在我们的数据中心或托管设施中管理和操作全栈 HCI。

在本地全栈基础设施上运行的工作负载享有多种优势 (sysin)，包括安全性、治理、低延迟、性能、可预测的成本等等。不幸的是，这些工作负载无法获得云的弹性规模和核心优势，尤其是在大型全球分布式环境中。

新的云服务

VMware Cloud Foundation+ 将通过简化的订阅模式和无密钥授权提供新的管理员、开发人员和混合云服务。这些新的云连接服务将在您的整个环境中实现全球库存的可见性、简化的维护、统一的安全性等等。

借助这些新功能，组织可以通过在混合云和多云部署中对基于 VM 和容器的企业工作负载进行简化的全局管理来实现更高的运营效率 (sysin)。此外，客户将能够简化维护窗口、改善安全状况并立即访问新功能和云服务。

由 VMware Cloud Foundation 4.5 提供支持

VMware Cloud Foundation 4.5 将通过添加 vSphere+ 和 vSAN+ 以及一个云网关来启用 VMware Cloud Foundation+，该网关作为完整堆栈架构的一部分提供对 VMware Cloud Console 的访问。VMware Cloud Foundation 4.5 的目标是在 VMware 2023 财年第三季度（2022 年 10 月 28 日）结束之前发布 (sysin)，还包括迁移、可扩展性和可用性方面的关键增强功能。运行 VCF 4.0 或更高版本的现有客户可以将其现有部署升级到 VCF 4.5，这可以启用云网关并提供对 Cloud Console 的访问。

独特的 VMware Cloud Foundation+ 云连接服务

借助 VMware Cloud Foundation+，您将可以访问以下新服务：

库存服务：

  • 提供任何 VMware Cloud Foundation+ 和 vSphere+ 连接实例的高级全局清单视图。

生命周期服务：

  • 如果 VMware Cloud Foundation 实例中有可用更新，则向管理员提供通知。实际的生命周期管理由 SDDC-manager 实例直接执行。

虚拟机用户服务：

  • 这允许管理员将虚拟机从云中部署到任何连接的 vCenter 上，就像现在的 vSphere+ 一样运行。

基础设施运营服务：

  • Infra Ops 服务通过提供跨多个中心和跨 VCF 工作负载域、vSphere+ 和 vSAN+ 的集群的整合事件、安全状态、容量状态的视图，提供所有 VCF 实例的全局概览。

注意：对于部署必须断开连接的客户或运行早期 VMware Cloud Foundation 3.X 版本的客户，VMware Cloud Foundation-Subscription (VCF-S) 可提供 VMware 的完整堆栈功能和规模 (sysin)，以及一个期限的成本优势订阅优惠。

VMware Cloud Foundation+ 将开启混合云服务的新时代，在单一解决方案中充分利用本地和云连接的基础架构。所提供的服务范围将使管理员和开发人员能够访问一个全面的工作负载平台，该平台支持虚拟机和基于容器的应用程序的整个生命周期。借助这种新的 SaaS 模式，其他混合云服务将以更快的速度交付，并通过简化的订阅模式和无密钥授权启用。

什么是 VMware Cloud Foundation？

VMware Cloud Foundation 是基于全栈超融合基础架构 (HCI) 技术构建的混合云平台，用于管理虚拟机和编排容器。凭借易于部署的单一体系架构，VMware Cloud Foundation 可在私有云和公有云之间实现一致、安全的基础架构和运维 (sysin)。通过可以提供这一切的混合云，提高企业的敏捷性和灵活性。

借助适用于现代应用的混合云平台，推动实现数字化转型。

现已提供集成式容器编排和 VMware Tanzu 管理工具。

VMware Cloud Foundation 的优势

VMware Cloud Foundation 可大幅缩短实现真正的混合云的过程，从而提高管理效率并降低总体拥有成本。

  • 简化的管理

    通过集成的容器编排由 vSphere 7 with Kubernetes 提供支持）优化 Kubernetes 集群的性能、恢复能力和可用性。

  • 灵活的部署选项

    跨私有云和公有云访问极为广泛的部署选项，并从自管理 HCI 和 HCI 即服务选项中进行选择。

  • 增强的应用性能

    通过高性能存储和跨每个环境的无缝资源池的负载均衡 (sysin)，提高应用的性能和可靠性。

  • 标准化体系架构

    使用您在数据中心使用的一流的计算、存储、网络连接和管理工具自动执行 HCI 交付，以简化运维并降低 TCO。

  • 最大化规模和 IT 效率

    通过在各处扩展相同的基础架构、运维、工具和流程，管理本地部署或云中的工作负载。

  • 原生安全性

    通过我们的全栈 HCI 平台确保企业级安全性，该平台整合传统虚拟机和现代容器工作负载。

VMware Cloud Foundation 的应用场景

  • 私有云和混合云

    从本地部署、边缘或云环境中的虚拟化资源池调配基础架构 (sysin)。自动化和编排简化了基于虚拟机或容器的工作负载的管理，这些工作负载可以扩展以满足您的业务需求。

  • 现代应用

    通过 VMware Cloud Foundation Services（一种集成了 Kubernetes 和 RESTful API 的新界面）在本地或云中开发和部署现代应用。

  • 虚拟桌面基础架构

    实现 Horizon 7 虚拟桌面基础架构 (VDI) 的自动化部署，以标准化、自动化和简化虚拟桌面。

版本比较

选择最适合您的 VMware Cloud Foundation 版本。

Starter：自动执行基础架构部署和调配；包括针对传统虚拟机工作负载的 VMware 核心超融合体系的生命周期管理。

Standard：在本地部署数据中心采用云计算管理功能，将 Kubernetes 编排与虚拟机管理集成。

Advanced：利用高级云计算管理工具以及集成的 Kubernetes 编排和工作负载自动化，改进基于 VMware 的云运维模式。

Enterprise：使用高级管理、网络连接和存储实施完整的企业云运维模式（通过 Kubernetes 编排和工作负载自动化进行了优化）。

常见问题解答

查找有关 VMware Cloud Foundation 的一些常见问题的答案

  • 哪些版本的 VMware Cloud Foundation 包含集成的 vSphere with Kubernetes？

    Standard、Advanced 和 Enterprise 版 VMware Cloud Foundation 包含 vSphere 和 Tanzu Kubernetes Grid 功能。VMware Cloud Foundation Starter 版包含标准 vSphere 虚拟机功能。

  • 何时支持 VMware Cloud Foundation 早期版本的升级？

    VMware Cloud Foundation 4.0 仅限于全新 VCF 环境 (sysin)。目前不支持对现有 VCF 部署进行升级。现有的 VMware Cloud Foundation 实例可以从 3.9 升级到 3.9.1，在未来版本中还将支持升级到 4.x。

  • 如何将 VMware Cloud Foundation 用于混合云？

    VMware Cloud Provider 计划 (VCPP) 中的精选服务提供商提供由 VMware Cloud Foundation 支持的云服务，这些提供商包括 CenturyLink、OVH 和 Rackspace。VMware Cloud on AWS 是 VMware 运维、管理和销售的按需服务，由 VMware Cloud Foundation 提供支持。其他解决方案包括 Azure VMware Solutions by CloudSimple 和 Google Cloud VMware 解决方案。

  • 如何购买 VMware Cloud Foundation 软件？

    购买 VMware Cloud Foundation 有四种方式：(1) 直接从 VMware 购买；(2) 通过 VMware 渠道合作伙伴购买；(3) 作为集成系统的一部分通过 OEM 供应商购买；以及 (4) 作为订阅服务通过公有云服务提供商购买。
