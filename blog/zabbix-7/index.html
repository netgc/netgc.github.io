Zabbix 7.0 新特性

Zabbix 7.0 LTS版本是基于GNU Affero通用公共许可证第3版（AGPLv3）发布的，7.0 LTS更新了许多新的功能，包含合成终端用户Web监控、Zabbix proxy高可用性和负载均衡、重大性能和可扩展性提升、原生多因子认证（MFA）支持、多种新方法可视化数据和基础设施状态，以及其它各种功能！

1、合成终端用户Web监控

通过定义灵活的基于场景的多步骤监控网站和Web应用程序。

  • 捕获当前网站状态的截图
  • 采集并可视化网站性能和可用性指标
  • 提取和监控Web应用程序中的任何数据
  • 分析采集的数据并接收与发现的问题相关的警报

2、Zabbix proxy高可用性和负载均衡

通过自动proxy负载均衡和高可用性功能，按需扩展Zabbix环境并确保100%的可用性。

  • 将主机分配给负载均衡的proxy组
  • 通过部署新的proxy无缝扩展Zabbix环境
  • Proxy和proxy组的数量没有限制

](https://assets.zabbix.com/img/7.0/proxy_group_list_1100x475.png)

创建负载均衡的proxy组

3、更快更高效的Zabbix proxy

Zabbix proxy现在支持将所有采集的指标存储在内存中。

  • 可选择磁盘、内存和混合proxy缓冲模式
  • 内存模式支持边缘计算用例
  • 根据分配的硬件，proxy性能提升高达10到100倍
  • 非常适合嵌入式硬件

​

4、提升数据采集速度和扩展性

为了大幅提升指标轮询的速度和扩展性，同步式poller进程已被异步式poller取代。

  • 更快的agent、SNMP和HTTP检查指标轮询
  • 在等待先前请求的指标响应之前，现在可以轮询下一个指标
  • 每个poller进程支持高达1000个并发检查

​

5、集中控制数据采集超时

集中控制数据采集超时可以更好地处理较长数据采集时间间隔的指标和自定义检查。

  • 监控项级别定义超时时间
  • Proxy级别覆盖超时时间
  • 完全可以通过Zabbix前端页面或Zabbix API配置超时时间



直接从前端页面控制数据采集超时时间

7、显著提升网络自动发现速度

网络自动发现支持并行检查，将主机和服务发现的速度提高了10到100倍。

  • 网络自动发现现在支持网络发现规则内的并行检查
  • 并发支持使得网络自动发现速度得到显著改善
  • 网络自动发现错误现在在前端显示

使用并发网络自动发现，只需几秒钟就可以发现主机

8、动态仪表板小部件导航

为仪表板小部件引入了一个新的通信框架，实现了小部件之间的通信。

  • 一个小部件可以作为其它小部件的数据源，例如，在地理地图或网络拓扑图上显示所选主机的信息
  • 仪表板小部件中显示的信息会根据数据源动态更新
  • 为你的使用案例构建自定义交互式 Zabbix 仪表板

利用小部件作为数据源创建动态仪表板

9、企业级多因子认证支持

开箱即用的多因子认证（MFA）支持提供了企业级安全性，并为配置用户认证方法提供了更大的灵活性。

  • 基于时间的一次性密码（TOTP）认证
  • Duo通用提示认证

10、更灵活的资源发现和管理

低级别自动发现已经进行了各种改进，使得在发现复杂环境中的主机（如VMware或Kubernetes）时，能够实现增强的主机配置和管理灵活性。

  • 能够将低级别自动发现的主机链接到其它低级别自动发现规则创建的组中
  • 能够自动禁用丢失的资源
  • 将默认的低级别自动发现规则更新间隔更改为1小时

选择当发现的资源被标记为丢失时的行为

12、其它的变化和提升

  • Zabbix许可模型已更新为GNU Affero通用公共许可证第3版（AGPLv3）
  • 通过重新设计底层权限检查逻辑，改进了Zabbix前端性能
  • 更快地重新计算主机维护状态
  • 新的history.push API方法，用于通过Zabbix API将数据发送到Zabbix server
  • 支持前端脚本的手动用户输入
  • 大多数前端表单都在模态窗口中打开
  • 提供了用于将数据流式传输到Kafka代理的Kafka官方连接器
  • 在新安装中，将auditlog表转换为TimescaleDB上的hypertable，以从自动分区（默认为7天）和更好的性能中受益
  • 添加了在处于非活动模式的7.0 LTS agent上执行远程命令的能力
  • 为监控项、小部件、模板、审计日志和其它Zabbix功能进行了各种易用性改进
  • 支持生成多页定时报表，适用于多页仪表板
  • 通过“系统信息”部分/小部件引入了可选的Zabbix server和前端更新检查
  • 自动注册和网络自动发现操作现在支持添加和删除主机标签
  • 扩展了宏函数的支持
  • 现在支持在监控项名称和监控项原型名称中使用用户宏
  • 通过预处理改进了错误处理。在预处理步骤中可以使用正则表达式匹配错误消息
  • 每个标准监控项现在都有从前端到其文档页面的直接链接
  • 提升了各种函数行为
  • 触发器表达式和可计算监控项支持jsonpath()和xmlxpath()函数
  • 新增net.dns.perf和net.dns.get监控项
