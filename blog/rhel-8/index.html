红帽企业 Linux 8.10 新增功能

使用红帽企业 Linux 8.10 优化应用程序生命周期

2024 年 5 月 23 日

定义应用程序生命周期可以为您提供一种更加结构化的方法来设计、构建和交付满足用户需求和业务目标的应用程序。从最初的开发到生产部署，再到持续的管理和优化，精心规划的应用程序生命周期可帮助您提供随着时间的推移保持更具适应性和弹性的解决方案。借助红帽企业 Linux (RHEL) 8.10 的新功能，您可以在值得信赖且更可靠的软件基础上更好地加速和简化应用程序生命周期，包括开发、部署和管理任务。

开发创新的新解决方案

构建和部署现代云原生应用程序可以帮助您增强客户体验、简化运营并提高敏捷性。借助 Application Streams 中提供的先进技术，您可以开发新的创新应用程序 (sysin)，使您的业务脱颖而出，帮助您占领新的市场份额并在竞争中保持领先地位。RHEL 8.10 中的新应用程序流包括：

  • PHP 8.2 。通过 PHP 语言的这一重大更新，提高应用程序性能和稳定性，其中包括只读类、面向对象的随机数生成器应用程序编程接口 (API) 和其他独立类型等新功能。
  • Python 3.12 。通过新的优化和功能简化开发流程，例如灵活的 f 字符串解析、用于调试和分析的新 API，以及对隔离子解释器和跟踪中的 Linux 性能分析器的支持。
  • PostgreSQL 16 。通过添加的功能和增强功能加速数据库工作负载 (sysin)，包括并行散列连接、备用服务器的逻辑复制、并行大型事务、改进的统计监控以及新的 SQL/JSON 构造函数和身份函数。
  • Ruby 3.3 。通过 Prism 解析器和纯 Ruby 即时 (JIT) 编译器等新增强功能减少应用程序内存使用和运行时间。
  • MariaDB 10.11 。使用新的服务器和管理员身份验证功能保护敏感数据，并通过优化器和复制改进提高应用程序性能。
  • nginx 1.24 。 通过改进的标头处理和优化的内存使用等新功能增强内容和应用程序交付。
  • 编译器工具包。使用新工具版本简化应用程序开发，包括 LLVM 17 、 Rust 1.75 和 Go 1.21 。

随处构建和部署应用程序

容器可简化应用程序部署，并有助于在不同的混合云环境中提供更高的一致性和灵活性。是一项关键的容器技术，Podman 是一个用于开发、管理和运行容器的开源工具。Podman 4.9 （包含在 RHEL 中）有众多新功能。

  • 场功能简化多架构容器构建 使用 多架构构建 ，该功能允许您将构建分派到不同平台上的 Podman 主机。
  • 简化复杂标志集的管理。 通过将多个命令行选项组合在一个可重用的模块中，
  • 语法简化容器文件 使用 Heredoc 以包含简短的脚本和文件，并使用 Podman REST API 查看部分进度报告。
  • 提高容器的恢复能力。 使用 SQLite（新安装的默认数据库）发生异常终止时，
  • 自动部署生产就绪的容器工作负载， 使用 Podman 系统角色在边缘设备上 Podman 之一。系统角色是简化跨现场基础设施、云资源和边缘设备的 RHEL 系统管理的众多系统角色

提供更加一致、可靠的用户体验

通过自动化基础设施配置任务，关键工作负载和应用程序可以不间断地运行，并具有更高的一致性、更好的可靠性和更高的安全状况。

  • 借助 引导加载程序 系统角色，您可以自动配置内核命令行参数、超时和密码 (sysin)，以提高一致性并大规模简化系统管理。
  • 系统角色 ad_integration 可让您在与 Microsoft Active Directory 集成的 RHEL 主机上更灵活地配置动态域名系统 (DNS) 更新。
  • 使用 fapolicyd 部署和配置来提高混合云安全性 系统角色，您可以通过自动 fapolicyd ，以允许或拒绝大规模应用程序执行。

当发生故障时，快速的数据和系统恢复对于最大限度地减少中断、恢复业务运营和减轻潜在损失至关重要。

  • 新的 快照 系统角色通过自动创建和管理逻辑卷管理器 (LVM) 存储卷的时间点快照，提供更快、更可预测和可重复的大规模数据备份和恢复。
  • 通过 高可用性集群 系统角色，您可以配置仲裁设备 (qdevice) 选项和防护拓扑，以便在系统无响应时提供额外的灵活性。
  • 如果 SAP HANA 索引服务器发生故障，RHEL 高可用性附加组件中的新功能会自动将进程切换到辅助节点，以缩短恢复时间并提高数据可用性。

将操作系统基础的生命周期延长 5 年以上

RHEL 的所有主要版本都遵循标准的 10 年生命周期。在 8.10 版本之后，RHEL 8 进入未来 5 年的维护支持阶段 (sysin)。在此期间，红帽将提供根据 RHEL 维护支持阶段政策发布的技术支持、安全补丁和错误修复，让您可以继续使用 RHEL 8.10 直至 2029 年。

但是，当您准备从 RHEL 8 迁移时，红帽已准备好提供帮助。就地升级工具和详细指南可以简化最复杂的操作系统和应用程序迁移。通过与红帽咨询合作，您可以与专家团队合作规划和执行升级项目。

关于红帽企业 Linux

什么是红帽企业 Linux？

红帽企业 Linux 是企业 Linux 操作系统，已获得数百个云服务及数千个供应商的认证。红帽企业 Linux 能够跨环境提供一致的基础，并提供帮助任何应用快速交付服务和工作负载所需的工具 (sysin)。红帽企业 Linux 可减少部署摩擦，降低成本，同时加速关键工作负载实现价值的时间，使开发和运维团队能够在任何环境中一起创新。
