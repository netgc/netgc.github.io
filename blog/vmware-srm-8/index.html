什么是 Site Recovery Manager (SRM)？

Site Recovery Manager 是业界领先的灾难恢复 (DR) 软件，可自动编排故障转移和故障恢复，以最大程度地减少停机。内置的无中断测试和报告功能可简化审核，并确保满足您的恢复时间目标 (RTO)。

了解如何开始体验这款简单、可靠的灾难恢复软件。

使用 SRM 针对私有云和公有云实现应用可用性和移动性

使用可在任何存储平台上运行的、与 VMware 集成的唯一解决方案简化灾难恢复。VMware Site Recovery Manager (SRM) 提供基于策略的管理，通过自动化编排最大限度缩短发生灾难时的停机时间，并支持对灾难恢复计划进行无中断测试。Site Recovery Manager 可作为永久许可证提供，或作为通过 VMware Site Recovery for VMware Cloud on AWS 提供的服务提供。

  • 基于策略的简单管理

    使用策略驱动的自动化功能，通过从 vSphere Web Client 托管的集中式恢复计划，轻松保护数千台虚拟机。

  • 混合云就绪

    通过适用于 VMC on AWS 的 VMware Site Recovery，实现端到端灾难恢复保护即服务。该快速部署解决方案可利用现有投资 (sysin)，让您为过渡到混合云做好准备。

  • 与任何存储兼容

    通过与所有主要 VMware 存储合作伙伴的 vSphere Replication、Virtual Volumes (vVols) 和基于阵列的复制解决方案进行原生集成，体验灵活性与自由选择权。

  • 降低 TCO

    降低灾难恢复的运维费用，并通过自动故障转移报告功能精简耗时的灾难恢复审核流程。

常见问题解答

什么是 Site Recovery Manager？

Site Recovery Manager (SRM) 是业界领先的灾难恢复 (DR) 管理解决方案，旨在最大限度地减少灾难情况下的停机。它为集中式恢复计划提供基于策略的管理、自动化编排和无中断测试。它专为虚拟机而设计，并可进行扩展，以在 vSphere 环境中管理所有应用。

SRM 可自动执行故障转移和故障恢复并对其进行编排 (sysin)，确保在发生灾难时最大限度减少停机。内置的无中断测试可确保满足您的恢复时间目标。

总体而言，SRM 通过自动化简化了管理，确保了快速且高度可预测的恢复时间，并将总体拥有成本降至最低。SRM 可应对多种应用场景，例如灾难恢复、灾难规避、计划的数据中心迁移、站点级负载均衡，甚至应用维护测试。

什么是 vSphere Replication？

vSphere Replication 是 VMware 针对 vSphere 虚拟机开发的基于 hypervisor 的复制技术。vSphere Replication 可确保虚拟机免受部分或全部站点故障的影响。它通过与存储无关且以虚拟机为中心的复制简化了灾难恢复保护。它还支持具有多个时间点恢复和可自定义恢复点目标（从 5 分钟到 24 小时）的各种基础存储解决方案。

vSphere Replication 基于每台虚拟机进行配置，从而可以精细控制要复制的虚拟机。从主站点到辅助站点、单个站点中的两个集群之间，以及从多个源站点到单个目标站点会发生这种复制，而与基础存储阵列无关。vSphere Replication 是 vSphere 平台的一种功能特性，与 Site Recovery Manager 原生集成。

Site Recovery Manager 如何工作？

为获得最大的灵活性与自由选择权，Site Recovery Manager 通过 VMware Server 和基础复制技术与 vSphere 集成。它可以与 vSphere Replication、vSphere Virtual Volumes (vVols) 集成存储阵列进行原生集成，也可以与领先存储供应商通过存储复制适配器提供的各种基于存储阵列的复制解决方案进行原生集成。

SRM 指导用户完成配置恢复计划的过程。在故障转移或测试时，SRM 自动执行恢复计划。SRM 使团队即使在工作时间也可以执行频繁的无中断测试，以确保可预测的恢复目标。通过自动化编排工作流，可将恢复时间减少到几分钟。通过编排跨站点的大规模虚拟机实时迁移，可在应用移动期间实现零中断。

Site Recovery Manager 部署需要哪些组件？

受保护站点和恢复站点均需要 vSphere、vCenter Server 和 Site Recovery Manager 的实例。SRM 还需要一款基础复制产品才能将虚拟机复制到恢复站点。客户可以选择使用 vSphere Replication、Virtual Volumes (vVols) 集成存储阵列或基于第三方阵列的复制软件。使用基于阵列的复制软件时，需要存储复制适配器 (SRA)。

支持哪款复制软件？

Site Recovery Manager 要求对 iSCSI、FibreChannel 或 NFS 存储阵列使用 vSphere Replication、vSphere Virtual Volumes (vVols) 集成存储阵列、基于阵列的复制或基于存储的复制。对于基于存储的复制，VMware 与存储合作伙伴合作，确保客户可以使用自己选择的存储和存储复制平台来部署 SRM。SRM 通过存储复制适配器 (SRA) 插件与各种复制软件一起工作，这些插件由存储供应商开发并通过认证，以便与 SRM 一起使用。当使用 vVols 集成存储阵列时，VASA 提供程序将替换 SRA。有关 SRA 和受支持存储的最新列表，请查看 存储合作伙伴兼容性列表。查看 SRM 以了解 vVols 兼容性列表。

Site Recovery Manager 和 VMware Site Recovery 有什么区别？

VMware Site Recovery for VMware Cloud on AWS 是一项单独的服务，但基于 Site Recovery Manager 和 vSphere Replication 而构建。Site Recovery 提供按需灾难恢复即服务，以保护本地部署和 VMware Cloud on AWS 上的工作负载。该服务在本地部署数据中心与 VMware Cloud on AWS 之间，以及 VMware Cloud on AWS 的不同实例之间的灾难恢复事件中自动恢复工作负载。它可将工作负载复制到 VMware Cloud on AWS，一旦您需要，可以随时启动这些工作负载。

VMware Site Recovery 如何更快地提供保护？

VMware Site Recovery 可通过利用云灵活性和经济性，更快地提供保护。您可以轻松地在 AWS 上启动 VMware 环境，而无需构建辅助本地部署灾难恢复站点。Site Recovery 还使用同样熟悉的工具和运维环境（从本地部署到公有云），极大地简化了灾难恢复运维，并实现“一天完成灾难恢复”。
