VMware Aria 介绍

​普尼玛帕德马纳班

2022 年 8 月 30 日

云原生应用和多云的统一管理解决方案

我对本周在 VMware Explore 上再次见到我们的客户的前景感到非常兴奋，尤其是在我们将宣布 VMware 云管理故事的下一章之际。我们将推出 VMware Aria，这是一个多云管理产品组合，它提供了一组端到端解决方案 (sysin)，用于管理基础架构和应用程序的成本、性能、配置和交付。VMware Aria 专为应对云原生应用程序和公共云环境的运营挑战而设计，真正为多云管理提供了全新的视角。

今天，我们越来越多的客户正在跨公共云和边缘交付应用程序，他们越来越多地使用 Kubernetes 来管理和扩展他们的现代化和云原生应用程序。为了成为客户首选的多云管理合作伙伴，我们需要解决他们在管理任何云的整体云支出、资源利用率以及应用程序性能、安全性和合规性方面面临的挑战。VMware Aria 使我们能够做到这一点。

为 VMware Aria 提供支持的是 VMware Aria Graph，这是一种令人兴奋的全新云规模数据存储技术，可在单个视图中捕获和映射客户的多云环境（应用程序、用户、配置和相关依赖项）的复杂性。创新的 Aria Graph 技术巩固了我们所有的核心管理解决方案，即 Aria Cost、Aria Operations 和 Aria Automation。此外，它还为大规模的云治理 (Aria Guardrails)、云迁移 (Aria Migration) 和业务洞察 (Aria Business Insights) 提供了一套端到端多云管理解决方案 (sysin)。VMware Aria Graph 将作为 VMware Aria Hub 中的一项功能提供给客户。VMware Aria Hub 首次在 VMworld 2021 上作为 Project Ensemble 推出，提供集中的视图和控件来管理整个多云环境。

​

可扩展性和可扩展性是 VMware Aria 解决方案的核心原则。VMware Aria 能够可靠地扩展到数亿个对象并在它们出现时摄取数十亿个数据点，从而满足最大型和最苛刻企业的多云管理要求。VMware Aria 的图形数据存储和 API 服务使其能够与第三方解决方案无缝集成，例如可观察性和应用程序性能管理工具。Aria Graph 中从本地公共云、本地云、VMware 工具和第三方工具收集和规范化的数据丰富了客户现有 VMware vRealize、CloudHealth by VMware 和 VMware Tanzu Observability 产品中的运营、自动化和成本指标。

VMware Aria 旨在解决客户在继续采用原生公共云和云原生应用程序时越来越多地遇到的新兴跨云和跨学科管理挑战。一个很好的例子是云迁移。对于那些拥有多个云环境并寻求对其现有产品组合进行现代化或合理化的客户来说，他们现在可以在迁移应用程序或工作负载时选择各种目标云选项 (sysin)。从成本、性能和安全角度了解目标云的特征并将其与其他云选项进行比较取决于拥有跨这些云环境的所有对象和工作负载的关系图。这正是 VMware Aria 的新数据存储 - VMware Graph - 的设计目的。VMware Aria 不是关于管理过去的挑战，而是关于管理多云和云原生应用程序世界不断增长的复杂性。

随着 VMware Aria 的推出，我们在四个关键方面统一了 VMware Cloud Management：

  • 产品：VMware Aria - 一个多云管理产品组合，用于管理跨任何云的基础架构和云原生应用程序的成本、性能、配置和交付
  • 平台：由 VMware Aria Graph 提供支持的 VMware Aria Hub - 由基于图形的数据存储提供支持的集中视图和控件，可捕获多云环境的资源和关系
  • 解决方案：新的端到端服务 - 三个新的端到端解决方案跨越管理学科，以提供云治理、云迁移和业务洞察力
  • 产品组合：VMware Aria - 统一 VMware 产品组合的单一家族名称：vRealize、CloudHealth by VMware Suite 和 Tanzu Observability

我要感谢所有客户的支持，因为我们共同构建了云管理的未来 - VMware Aria。

VMware Aria 应用场景

  • 安全配置

    增强云工作负载、服务和基础架构的安全性与合规性 (sysin)。这使 IT 团队能够更有效地管理风险，在云环境内和跨云环境进行监管，并确保多云环境保持合规性。

  • 加速自动化

    利用可跨多云环境提供自助式使用体验的现代开发人员平台加快服务交付速度。

    为可支持基础架构即代码 (IaC) 和流水线功能的基础架构提供 DevOps。通过中央管理平面实现 Kubernetes 自动化和管理。

  • 控制成本和容量

    通过为希望持续分析云成本和容量趋势的团队提供财务管理和运维功能，简化云成本和容量管理。

    按成本中心监控、分析和报告云支出 (sysin)，根据预算提高成本效益和责任，并发现在不增加风险的情况下优化支出的机会。

  • 优化性能

    获得应用、基础架构和平台服务的统一可见性和一致运维。

    跨多个云环境、以及来自单一可信来源且基于容器的传统和现代应用，实现基于人工智能的持续性能优化、统一监控、迁移规划以及网络优化。
