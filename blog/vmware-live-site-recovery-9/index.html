VMware Live Recovery

通过跨混合云的统一管理体验大规模保护网络和站点恢复。

VMware Live Recovery 在基于云的单一界面中提供勒索软件和灾难恢复功能的统一管理。

VMware Live Recovery 为托管云或本地环境中基于 VMware 的服务器和工作负载提供针对现代勒索软件和灾难情况的保护，以确保关键资产的可用性和可恢复性。

VMware Live Recovery 使用单一部署方法和灵活的许可模式让数据保护和恢复变得更加轻松，因此您可以从一个界面调配和管理 VMware Live Cyber Recovery 和 VMware Live Site Recovery。

VMware Live Cyber Recovery
VMware Live Cyber Recovery 是一种易于使用的按需勒索软件和灾难恢复 (DR) 解决方案，以 SaaS 形式提供，具有云经济性。

VMware 勒索软件恢复在 VMware Cloud 恢复 SDDC 上提供隔离恢复环境 (IRE)，允许您检查、分析、恢复并验证受感染的虚拟机是否可以安全地将其恢复到生产环境。

VMware Live Site Recovery
VMware Live Site Recovery 是一种按需端到端灾难恢复解决方案 (sysin)，它使用基于阵列、基于主机的 VMware vSphere Replication 或 VMware 虚拟卷 (vVol) 复制以及 VMware Live Site Recovery 编排。

名称变更
随着 VMware Live Recovery 的首次发布，两个现有的 VMware 产品已更名：

旧名称 新名字
VMware Cloud Disaster Recovery, VMware Cloud DR VMware Live Cyber Recovery
VMware Site Recovery Manager, VMware SRM VMware Live Site Recovery

这两个 VMware 解决方案 VMware Live Cyber Recovery 和 VMware Live Site Recovery 现已成为 VMware Live Recovery 的一部分。

VMware Cloud Foundation 的网络和数据弹性

  • 统一防护

    通过大规模的全自动编排和统一的管理体验，跨本地和公共云保护和恢复虚拟机。

  • 加速恢复

    通过在按需隔离恢复环境 (IRE) 中对已启动工作负载进行实时行为分析，可以从现代勒索软件中可靠、快速地恢复。

  • 简化消费

    跨用例和云的灵活许可 (sysin)，并保证单一供应商支持。

VMware Live Cyber Recovery

以前称为 VMware Cloud Disaster Recovery 和 VMware Ransomware Recovery

  • 按需恢复环境

    通过由 VMware 保护、构建和管理的按需环境，实现勒索软件的安全、受控恢复。

  • 实时行为分析

    通过嵌入式下一代 AV 和开机工作负载的行为分析来识别无文件攻击。

  • 勒索软件恢复工作流程

    利用分步指导工作流程 (sysin)，将恢复点的识别、验证和恢复集成到单个 UI 中。

  • 按钮式虚拟机网络隔离

    恢复时将虚拟机 (VM) 彼此隔离，以防止勒索软件横向移动和生产环境再次感染。

  • 引导式还原点选择

    通过 VMDK 更改率和文件熵等洞察来通知候选还原点的选择。

  • 不可变的气隙恢复点

    将快照存储在 VMware 管理的安全云文件系统中，以在恢复时保持数据完整性。

VMware Live Site Recovery

以前称为 Site Recovery Manager

  • 快速可靠的恢复

    自动执行恢复计划并消除手动流程涉及的风险。

  • 零停机应用程序移动性

    灵活定制组织的拓扑和 RPO，同时执行恢复、大规模应用程序实时迁移和无中断测试。

  • 简化灾难恢复管理

    从设置到管理，借助自动化以及与 VMware Cloud Foundation 的集成 (sysin)，VMware Live Site Recovery 可为数千台虚拟机提供简单、基于策略的操作。

  • 频繁的无中断测试

    即使在工作时间内也执行频繁、无中断的测试，以确保可预测的恢复目标。

  • 自动化编排工作流程

    使用 VMware Live Site Recovery 的自动化编排工作流程，可靠地将恢复时间缩短至几分钟。

  • 大规模自动故障恢复

    通过自动故障恢复使用集中恢复计划，轻松返回常规操作。

新增功能

VMware Live Site Recovery（以前称为 Site Recovery Manager）9.0 中的新增功能

  • VMware Live Recovery

    一种新的解决方案，可在单个统一控制台中跨 VMware Cloud 提供勒索软件保护和灾难恢复 (sysin)。VMware Live Recovery (VLR) 旨在帮助组织保护其基于 VMware 的应用程序和数据免受各种威胁，包括勒索软件攻击、基础设施故障、人为错误等。有关详细信息，请参阅 VMware 实时恢复。

  • 每个保护组的最大虚拟机数量增加至 1500 个。

    Site Recovery Manager 9.0 将每个保护组的虚拟机数量增加到 1500 个。

  • 适用于 VMware Site Recovery Manager 9.0 的 VMware Aria Automation Orchestrator 插件

    有关新工作流程的信息，请参见 适用于 VMware Site Recovery Manager 9.0 的 VMware Aria Automation Orchestrator 插件发行说明。

  • 适用于 VMware Site Recovery Manager 9.0 的 VMware Aria 操作管理包

    有关管理包的信息，请参阅 适用于 VMware Site Recovery Manager 9.0 的 VMware Aria Operations Management Pack 发行说明。

兼容性

Site Recovery Manager 兼容性列表：

Site Recovery Manager 9.0 与 vSphere 7.0 Update 3 及更高版本兼容，并支持 ESXi 版本 7.0 Update 3 及更高版本。

Site Recovery Manager 9.0 要求受保护站点和恢复站点上都有受支持的 vCenter Server 版本 (sysin)。

有关互操作性和产品兼容性信息，包括对客户操作系统自定义的支持，请参见 VMware Site Recovery Manager 9.0 兼容性表。

兼容的存储阵列和存储复制适配器：

有关受支持的兼容存储阵列和 SRA 的当前列表，请参阅《 Site Recovery Manager 存储合作伙伴兼容性指南》 。

兼容的 Virtual Volumes 合作伙伴 VASA 提供商：

有关兼容 Virtual Volumes 合作伙伴 VASA 提供商的最新列表，请参阅 VMware 兼容性指南。

VMware vSAN 支持：

Site Recovery Manager 9.0 可以使用 vSphere Replication 保护驻留在 VMware vSAN 和 vSAN Express Storage 上的虚拟机。vSAN 不需要存储复制适配器 (SRA) 即可与 Site Recovery Manager 9.0 配合使用。
