概述

提供统一纳管、迁移和灾难恢复。

VMware Cloud Director Availability 是一款功能强大的解决方案，旨在“向”多租户 VMware 云环境或在这些云环境“之间”提供简单、安全且经济高效的纳管、迁移和灾难恢复服务。

优势

简单、经济高效的灾难恢复和迁移。

简单

实现基于熟悉工具而构建的一致管理功能，该管理方式综合了以前三个解决方案的功能：vCloud Availability for VMware Cloud Director 2.0、vCloud Availability Cloud-to-Cloud DR 1.5 和 VMware Cloud Director Extender 1.1。简单性则得益于现代化的 HTML-5 界面、与 VMware Cloud Director 的原生集成、新的快速设备部署模式以及面向租户和服务提供商的单个基于角色的访问控制 (RBAC) 门户。

经济高效

从基于订阅且价格具有竞争力的解决方案中受益，该解决方案设计的核心功能可最大限度降低成本，而紧密集成则可降低运维开销 (sysin)。通过源自 VMware vSphere Replication 的存储独立性、租户自助式保护、故障转移和故障恢复工作流以及基于虚拟机或 vApp 的精细控制，可提高灵活性并提供更具吸引力的经济效益。

集成

作为 VMware Cloud Provider 平台的一部分，VMware vCloud Availability 经过全新设计，极大地简化了云环境的纳管，实现了经济高效的可用性和恢复能力，并为云服务提供商及其终端客户提供安全运维。该解决方案与 VMware Cloud Director 紧密集成 (sysin)，可为基于 VMware Cloud Director 的云环境提供灾难恢复、纳管和迁移功能。

保护

由于具备 VMware 软件体系的内置安全性，包括对静态数据和动态数据的加密，因此用户可以自信地部署安全解决方案。此外，该解决方案还提供对复制流量的内置压缩以及端到端 TLS 加密。

新增功能

VMware Cloud Director Availability 4.6 现在支持以下新功能：

  • vSphere 到 vSphere 恢复计划

    • 恢复计划现在还可以为 vSphere DR 和 vCenter Server 站点之间的迁移创建和运行。

  • 审计日志

    • 事件和通知现在允许使用 Cloud Director 交付通道审核 VMware Cloud Director 可用性事件。VMware Cloud Director 自动将所有外部事件（例如从 VMware Cloud Director 可用性收到的事件）标记为审计事件，并通过使用 VMware Cloud Director 中的审计跟踪系统进行审计持久性、保留和导出功能。

  • 恢复设置增强

    • 现在，您可以为本地到云复制的每个选定复制映射源网络和目标网络，类似于从云到云的复制。有关详细信息，请参阅 配置本地到云复制的网络设置。
    • vSphere DR 和迁移现在根据复制设置（数据存储）验证恢复设置（数据中心、VM 文件夹、计算等）。网络设置现在类似于 Cloud Director 站点，源和目标网络映射允许每个虚拟机网络映射。有关详细信息，请参见 配置 vSphere DR 和迁移的恢复设置。

  • vSphere 到 vSphere DR 和迁移公共 API

    • 配置已部署设备的分步指南 现在，VMware Cloud Director Availability 提供了使用VMware Cloud Director Availability API：
    • API参考
      • 新增内容： VMware Cloud Director Availability 4.6 - vSphere DR 和 Migration API 参考
      • VMware Cloud Director 可用性 4.6 - Cloud Director DR 和迁移 API 参考
    • 节目指南
      • 新增内容： VMware Cloud Director 可用性 4.6 - vSphere 灾难恢复和迁移 API 指南
      • VMware Cloud Director 可用性 4.6 - Cloud Director 灾难恢复和迁移 API 指南
    • 在左窗格中的 Configuration 下，单击 OpenApi Client 然后选择一个 API 并查看 Program Guide。

  • NSX-T vApp 边缘支持

    受 VMware Cloud Director 10.3 或更高版本支持的 VMware Cloud Director Availability 4.6 现在支持：

    • 将路由的 vApp 网络和 vApp 网络服务复制到由 NSX 支持的虚拟数据中心，并提供无错误的自动目标网络映射。
    • 在 vApp 隔离网络上复制 DHCP 服务，从而与 NSX for vSphere 保持一致。
    • 要配置路由 vApp 网络并使用任何 vApp 网络服务，例如隔离 vApp 网络中的 DHCP，必须使用 边缘 集群配置包含组织的虚拟数据中心。
    • 有关详细信息，请参阅 配置复制到云的网络设置和 配置云到云复制的网络设置。

  • 来宾自定义全局设置

    • 影响所有复制的全局设置 - 用户是否需要在故障转移时手动激活或停用来宾自定义。有关详细信息，请参阅 复制源 VM 来宾自定义设置。

  • 隧道设备高可用性

    • 在 Cloud Director 站点中，现在第二个隧道设备可以在主动-主动模式下运行，以实现隧道服务的高可用性，适用于新部署和升级的部署。有关详细信息，请参阅 在 Cloud Director 站点中为 HA 添加第二个隧道设备。有关两个主动-主动隧道设备实例的信息，请参阅 Cloud Director 站点中的部署架构。

  • vApp 模板复制

    • 在 Cloud Director 站点之间复制 vApp 模板现在包括保护允许跟踪源更改的 vApp 模板，允许在源更改时创建新的目标模板版本或覆盖目标。

  • 改进的事件通知

    • 事件和通知现在还可以使用电子邮件传递通道发送复制管理事件。有关详细信息，请参阅 在 Cloud Director 站点中配置提供程序事件通知和 为 vSphere DR 和迁移配置提供程序事件通知。
    • 现在，通过配置电子邮件传送渠道，租户和供应商可以订阅每周摘要电子邮件。订阅者无需登录 VMware Cloud Director Availability 即可随时了解其复制发生的情况。有关详细信息，请参阅 订阅每周摘要电子邮件。
    • 在 Cloud Director 站点中，作为 提供者 ，您现在可以在管理界面中看到活动摘要报告。有关详细信息，请参阅 以提供者身份查看 Cloud Director 站点中的活动摘要报告。

  • vSphere 到 vSphere 带宽限制

    • 带宽限制现在也可以对 vSphere DR 和迁移应用限制。注意： 应用限制需要一个或多个外部 Replicator Appliance 实例。有关配置提供商站点限制的信息，请参阅 配置云站点的带宽限制。

  • VMC 数据引擎 现在允许创建回 本地 vCenter Server 的 迁移。有关所有用例的信息，请参阅 复制工作负载。
