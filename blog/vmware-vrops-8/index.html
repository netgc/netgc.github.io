概述

VMware vRealize Operations 可提供从应用到基础架构的自动驾驶式 IT 运维管理，以优化、规划及扩展 SDDC 和 VMware Cloud 部署，同时提供对多个公有云的可见性。在 AI 和预测性分析的支持下，它可帮助 IT 部门在一个统一的运维平台上轻松且不受干扰地进行生产运维 (sysin)。无论本地部署还是 SaaS，vRealize Operations 均可提供持续性能优化、高效容量管理、主动规划、智能修复和集成式合规性。

为何选择 vRealize Operations？

  • 来自市场领导者的成熟解决方案

    连续四年被 IDC 评为市场领导者，其所提供的针对要求最严苛应用的自动驾驶式运维功能，值得客户信赖。

  • 真正的云计算管理平台

    自动驾驶式运维与深层数据和市场领先的自动化相关联，因此 IT 团队可以满怀信心地根据业务目标准确优化性能和成本。

  • 卓越的混合云监控和故障排除功能

    基于与广泛的应用和基础架构生态系统相关联的结构化和非结构化数据，运行智能分析，能够帮助客户迅速解决问题。

  • 专为 SDDC 和 vSphere 构建的原生控制台

    客户青睐由同一工程团队设计且可与 vSphere、VMware Cloud on AWS 和 vSAN 顺畅协作的原生工具。

  • 无规则的运维敏捷性

    vRealize Operations 提供从本地部署到 SaaS 的多个部署选项 (sysin)，以支持业务创新，缩短销售就绪时间以及实现快速可扩展性。具有分布式和复杂环境的客户可以跨多个地理位置分散的数据中心以及跨多个 vRealize Operations 或 vRealize Operations Cloud 实例获得联合视图。

应用场景

持续性能优化

在运维和业务目标的驱动下，通过由实时预测性分析驱动的操作和 AI 实现自动均衡工作负载并主动避免争用，从而确保以最低成本获得所需的应用性能。将自动化目标驱动的工作负载均衡和优化应用于 VMware Cloud Foundation、vSAN 或 VMware Cloud on AWS。

  • 自动均衡工作负载

    根据 VMware Cloud Foundation、vSAN 或 VMware Cloud on AWS 上部署的虚拟机的业务和运维目标，自动并持续地跨集群均衡工作负载。优化工作负载均衡，以节约成本、提高性能、增强合规性或进行整合。对照定义的目标持续验证工作负载性能，将预测性分析应用于项目未来要求，自动均衡工作负载或将负载均衡安排在方便的维护时段内。

  • 在虚拟机的整个生命周期持续自动执行工作负载安置

    集成 vRealize Operations 和 vRealize Automation，或集成 vRealize Operations Cloud 和 vRealize Automation Cloud，用于初始安置和持续安置，以满足利用率和业务目标，共享通用构造以交付新一代云计算管理平台。

  • 基于主机的自动安置由运维和业务目标驱动

    跨主机创建安置区（不考虑集群边界），以根据软件许可证实施、等级和其他标签优化工作负载的安置和均衡。将业务目标应用到 VMware Distributed Resource Scheduler (DRS) 并自动执行 DRS 管理。

  • Predictive DRS

    通过将 vRealize Operations 的预测性分析功能与 VMware Distributed Resource Scheduler (DRS) 相结合来避免资源争用。分析并预测未来的需求或争用情况，并主动移动工作负载以避免问题。

集成式合规性

通过集成式合规性和自动偏差修复，降低风险并实施 IT 和监管标准。

  • VMware Cloud 配置与合规性

    使用即时可用的集群、主机和虚拟机合规性仪表盘，为 vSphere、vSAN、NSX、VMware Cloud Foundation 和 VMware Cloud on AWS 启用安全配置。

  • VMware Cloud 法规遵从性

    根据 DISA、FISMA、ISO、CIS、PCI 和 HIPAA 等通用模板，测量 vSphere、vSAN、NSX、VMware Cloud Foundation 和 VMware Cloud on AWS 合规性程度，或创建自定义合规性标准。

  • 自动化配置管理

    通过即时可用的工作流和 vRealize Orchestrator 集成自动修复偏差。

新一代运维平台

vRealize Operations 提供多种部署选项，以支持业务创新、缩短价值实现时间并快速扩展。

  • 可通过本地部署或作为服务提供

    vRealize Operations 可作为本地部署软件或云计算服务提供。名为 vRealize Operations Cloud 的云计算服务可提供与传统产品完全兼容的功能特性，但具有软件即服务的所有优势，例如降低运维开销、自动更新以及更快地获得新功能特性和创新。

  • 跨多个数据中心联合

    利用跨多个和地理位置分散的数据中心的联合视图在同一位置管理大型分布式复杂环境。

  • 连续可用性

    连续可用性提供了跨故障域延伸 vRealize Operations 集群的能力，这将有助于 vRealize Operations 集群在发生故障时继续运行，从而确保不间断的运维视图。

  • 可延展的开放式平台

    利用具有扩展能力和恢复能力的开放式可延展体系架构，管理复杂的大型异构混合环境。部署 VMware 及第三方硬件和应用供应商提供的特定领域的管理包。

容量及成本管理和规划

通过与成本分析相关的基于 ML 的实时容量分析降低成本、提高效率，改善利用率和整合，同时积极规划未来。

  • 主动式容量管理

    实时预测性容量分析基于容量使用情况和需求提供主动式警报，并根据可行的建议（包括回收、采购和云迁移规划选项）进行分配。

  • 回收和优化规模

    在保证业务和运维目标的前提下，回收超额配置的容量，优化虚拟机规模并加大整合。实时预测性容量分析基于容量使用情况和需求提供主动警报。这些分析还提供切实可行的建议（包括回收）以及计算和存储采购方案。

  • VMware Cloud 成本管理

    将容量分析与成本核算信息结合起来，轻松了解和跟踪运维效率和容量管理如何推动成本效益。定义和自定义成本动因，并采取措施优化容量和成本。了解总体拥有成本，获得精细的节约机会来管理 VMware Cloud Foundation 和 vSAN 成本，并获得反馈和计费报告。在本地获取有关 VMware Cloud on AWS 的成本和计费的见解，并通过 VMware 与 Cloud Health 的集成扩展公有云成本核算可见性。

  • 容量规划

    使用灵活的容量建模来开发跨超融合、混合和多云环境的资源策略和情景假设。高级容量规划能够为新工作负载的最佳拟合建模、规划硬件采购、比较跨多个数据中心的成本以及规划向 VMware Cloud on AWS 的迁移。评估在私有云中部署工作负载的成本，与其他公有云环境（如 AWS、Azure 和 Google Cloud）的成本进行比较。

智能修复

借助从应用到基础架构的统一可见性以及与指标和日志相关的切实可行的洞察信息，更快速地预测和预防问题，并进行故障排除。通过与 VMware Cloud Foundation、vSphere 7.0、vSAN、VMware Cloud on AWS 和多个公有云（如 AWS、Microsoft Azure 和 Google Cloud Platform）进行原生 SDDC 集成，来精简和集中化 IT 运维。

  • 对 Workbench 进行故障排除

    获得应用和基础架构运行状况的统一 IT 运维视图，以便进行异常检测和加快根本原因分析。直观了解和关联关键绩效指标、度量标准、事件和日志，并获得切实可行的工作流以快速排除故障。通过高级预测性分析实现性能问题的主动修复。洞察应用与基础架构的依赖关系，以简化变更影响分析和故障排除过程。

  • 跨 VMware Cloud 和原生公有云的统一可见性

    为 VMware Cloud Foundation、vSAN 和 VMware Cloud on AWS 环境获取即时可用且可自定义的仪表盘、报告和视图。自定义仪表盘、报告和视图，以创建独特的工作流，并在基础架构、运维和应用团队之间轻松共享。与通用的票证系统和协作工具集成，如 Service Now 和 Slack。原生集成 AWS、Azure 和 Google Cloud Platform 帐户及指标，以实现统一的可见性，加快决策制定和故障排除。

  • 全方位故障排除

    通过在环境中同时使用衡量指标和日志进行全方位故障排除，使故障排除更智能。vRealize Operations 与 vRealize Log Insight 集成以及 vRealize Operations Cloud 与 vRealize Log Insight Cloud 之间集成，可将结构化数据（例如衡量指标和关键绩效指标）与非结构化数据（例如日志文件）融合在一起，以便更快地分析根本原因。

  • 原生 VMware Cloud 集成

    运用并扩展 vCenter、vSAN、VMware Cloud Foundation 和 VMware Cloud on AWS 等 VMware SDDC 组件。全面了解 VMware Cloud Foundation 部署，自动均衡工作负载，优化性能，并管理容量和成本。规划向 VMware Cloud on AWS 的迁移并在部署后均衡工作负载，管理容量，以及获取有关计费和成本的洞察信息。

  • vSAN 的全球运维

    在 vCenter 用户界面中了解 vSphere 和 vSAN 环境概况，包括 KPI、关键警报、容量概述和运维见解及建议。在上下文中全面启动 vRealize Operations 仪表盘，以进行全栈故障排除以及容量和成本管理，包括剩余容量和时间、重复数据消除以及压缩节约和回收机会。借助高级故障排除、主动式警报以及从虚拟机到磁盘的可见性，实现对多站点和 vSAN 延伸集群的集中式管理。

  • 容器和 Kubernetes 管理

    即时可用且可自定义的仪表盘、报告、视图和警报可帮助运用 vSphere 7.0 with Kubernetes，从而提供对传统虚拟机和容器环境的完整可见性和控制。在 vCenter 中纳管主管集群、命名空间、POD 和来宾集群时自动发现它们的新构造。获取新的摘要页面，以监控 Kubernetes 构造的性能、容量、利用率和配置。管理和预测容量，以显示容量瓶颈并优化容器环境规模。此外，使用容器管理包来扩展原生部署容器的这种可见性。

  • 在本地监控常见的打包应用和操作系统

    使 IT 和应用所有者能够更快地确定问题类别和解决问题。vRealize Operations 可轻松自动发现虚拟机和应用，部署具有完整生命周期管理功能的代理，并为常见的打包应用收集监控和排除故障所需的指标。vRealize Operations 会映射应用和基础架构之间的关系，以提供全体系的可视化，并缩短确定性能和可用性问题根本原因的时间。
