关于 vSphere Bitfusion

VMware vSphere Bitfusion 共享图形处理单元 (GPU) 等加速器以提供共享网络可访问资源池，从而支持资源密集型人工智能 (AI) 和机器学习 (ML) 工作负载。vSphere Bitfusion 可在 AI 框架、云站点、网络以及虚拟机、容器和笔记本等环境中运行。

vSphere Bitfusion 的工作原理是什么？

Bitfusion 要解决的是以下问题：

  • 有些团队在利用机器学习进行研究时没有 GPU 可以用来加速学习速度 (sysin)；
  • 而另一些团队已经拥有了 GPU，但是他们没有完全利用 GPU 的处理能力，很多 GPU 资源在白白闲置。

Bitfusion 把 GPU 资源集中起来，组成 GPU 资源池，然后共享给大家使用，这个方案分为两部分：

  • Bitfusion 服务器：把 GPU 安装在 vSphere 服务器上 (要求 vSphere 7 以上版本)，然后在上面运行 Bitfusion Server (以 OVA 格式提供的 virtual appliance)，Bitfusion Server 可以把物理 GPU 资源虚拟化，共享给多个用户使用。
  • Bitfusion 客户机：Bitfusion Client 是运行在其他 vSphere 服务器上的 Linux 虚机 (要求 vSphere 6.7 以上版本)，机器学习 ML (Machine Learning) 工作负载运行在这些虚拟机上 (sysin)，Bitfusion 会把它们对于GPU 的服务请求通过网络传输给 Bitfusion Server，计算完成后再返回结果。对于 ML 工作负载来说，远程 GPU 是完全透明的，它就像是在使用本地的 GPU 硬件。

vSphere Bitfusion 4.5 的新增功能

4.5.4 的新增功能

  • 增加了每日 nodetool repair 扫描，可修复 Apache Cassandra 数据库可能出现的任何问题。

4.5.3 的新增功能

  • VMware vSphere 8.0 支持 vSphere Bitfusion。
  • 可以使用 vSphere Bitfusion 用户界面延长 vSphere Bitfusion 客户端的过期日期。
  • 增加了对 Red Hat Enterprise Linux 9.0 及更高版本的次要版本的支持。
  • 增加了对 Rocky Linux 8 的支持。
  • 增加了对 Rocky Linux 9 的支持。
  • 增加了对 NVIDIA Driver 525.85.12 的支持。
  • 增加了对 NVIDIA CUDA 11.5 和 11.5.2 的支持。
  • 增加了对 Pytorch 1.9 和 1.10 的框架支持。
  • 增加了对 NVIDIA A40 48GB PCIE 的硬件支持。
  • 增加了对 NVIDIA L40 48GB PCIE 的硬件支持。
  • 增加了对 NVIDIA A30 24GB PCIE 的硬件支持。
  • 增加了对 NVIDIA A10 24GB PCIE 的硬件支持。
  • 增加了对 NVIDIA A2 16GB PCIE 的硬件支持。

4.5.2 的新增功能

  • vSphere Bitfusion 客户端可以在 vSphere Bitfusion 命令行界面中显示其当前 GPU 分配和利用率。
  • 可以在运行 run 或 request 命令时标记 vSphere Bitfusion 客户端，这样可以更好地识别 vSphere Bitfusion 用户界面中的客户端。
  • 可以使用 vSphere Bitfusion 命令行界面续订 vSphere Bitfusion 服务器证书。
  • 可以使用 vSphere Bitfusion 用户界面续订 vSphere Bitfusion 客户端证书。
  • 增加了对 Ubuntu Linux 22.04 的支持。
  • 增加了对 SUSE Linux Enterprise Server 15.3 的支持。
  • 增加了对 Red Hat Enterprise Linux 7.9 及更高版本的次要版本的支持。
  • 增加了对 Red Hat Enterprise Linux 8.5 及更高版本的次要版本的支持。
  • 增加了对 NVIDIA Driver 470.129.06 的支持。
  • 增加了对 NVIDIA CUDA 11.3 和 11.4.4 的支持。
  • 增加了对 NVIDIA cuDNN 8.2.4 的支持。
  • 增加了对 PyTorch 1.2 - 1.8 的支持。
  • 增加了对 Tensorflow 1.15、2.2、2.3、2.4 和 2.6 的支持。
  • 增加了对 TensorRT 7.1.3、7.2.3 和 8.0.3 的支持。
  • 增加了对 PaddlePaddle 2.0.0、2.2.2 和 2.3.0 的支持。
  • 移除了对 CentOS 8 的支持。

4.5 的新增功能

  • vSphere Bitfusion 显示 vSphere Bitfusion 集群的内存利用率和内核利用率信息。
  • 将在 vSphere Bitfusion 4.5.0 中移除对 vSphere Bitfusion 2.x.x 客户端的支持。
